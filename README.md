# Tetris

Проєкт гри `Tetris` для `Arduino Uno` з використанням `2.4 TFT LCD Shield` на базі контролера `ILI9341`.

Проєкт орієнтований на учнів віком `10-13 років`, тому структура репозиторію зроблена простою, модульною та зрозумілою. Код поділено на окремі частини:

- робота з дисплеєм через `MCUFRIEND_kbv`;
- робота з резистивним тачскріном через `TouchScreen`;
- логіка гри `Tetris`;
- рендер поля з частковим оновленням клітинок;
- конфігурація екрана, тачскріна і таймінгів.

## Мета проєкту

Створити навчальний і практичний приклад гри `Tetris`, який:

- працює на `Arduino Uno`;
- використовує готові бібліотеки з екосистеми `Arduino IDE`;
- показує принцип поділу програми на незалежні модулі;
- підходить для подальшого розширення.

## Апаратна платформа

- Плата: `Arduino Uno`
- Екран: `2.4 TFT LCD Shield`
- Контролер дисплея: `ILI9341`
- Ввід: `Resistive Touchscreen` на шилді
- Фреймворк: `Arduino`
- Основні бібліотеки: `Adafruit_GFX`, `MCUFRIEND_kbv`, `TouchScreen`

## Базова ідея архітектури

Проєкт базується на наступних принципах:

1. `HAL / Drivers`
   Відповідає за роботу з TFT-дисплеєм і резистивним тачскріном.

2. `Game Core`
   Містить логіку `Tetris`: поле, фігури, рух, перевірку ліній, рахунок, стани гри.

3. `Application Layer`
   Керує запуском гри, сенсорним вводом, оновленням стану й рендером.

4. `GameBoy.h`
    Забеспечуются суміснисть з інтерфейсом `GameBoy.h`. Реалізовані функціальні компоненті додаються до файлу бібліотеки `GraphBoy.h` с таким ж назвами функцій, які далі використовуються на наступних етапах проекту. Це забеспечує суміснисть з питаннями тесту курсу.

## Бібліотеки

Проєкт зараз налаштований під такі бібліотеки:

- `Adafruit GFX`
- `MCUFRIEND_kbv`
- `TouchScreen`

Перед запуском на реальному shield варто перевірити:

- які піни реально використовує ваш шилд;
- чи коректно читається `controller ID`;
- чи правильно відкалібровані значення тачскріна;
- скільки оперативної пам'яті займають шрифти, буфери й графіка.

## Поточне керування

- кнопка `LEFT` на лівій боковій панелі: рух ліворуч;
- кнопка `RIGHT` на правій боковій панелі: рух праворуч;
- кнопка `RESET GAME` на правій боковій панелі: перезапуск;
- центральна зона поля: поворот;
- нижня частина поля: прискорене падіння.

## Принципи для подальшої розробки

- не змішувати логіку гри з кодом малювання;
- не прив'язувати ядро гри напряму до конкретної бібліотеки дисплея;
- тримати код простим для читання учнями;
- додавати короткі зрозумілі коментарі;
- створити набір бонусів для учнів з додатковим "фішками"; 
- розширювати функціональність через окремі модулі, а не через один великий файл.
